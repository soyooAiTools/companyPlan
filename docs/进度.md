## 2026-06-17 · 环节改用 segment_id 表关联 ✅
**问题**:工单列表「环节」列显示旧名(如「程序1」),改名后不更新 —— 工单存的是环节名「快照」(`discipline`),没按 id 关联;改名同步又因当时打在旧后端实例上没生效,成了孤儿。
**改法**:`tickets` 加 `segment_id`(Prisma 标量 + `prisma generate`)→ `createTicket` 存 segment_id → `mapTicket` 的 `tagName` 改为按 `segment_id` 取**当前**环节名(回退 `discipline` 快照)。存量工单按环节名/标签回填 segment_id,孤儿单独修。
**效果**:改环节名 → 列表即时反映,不再依赖快照。
**验证**:三条工单 segment_id 已回填;孤儿「程序1」→ 当前名「程序」;mapTicket 解析实测正确。
**遗留**:交付时长/预警阈值仍是建单快照(改环节配置不回填老单),要跟随同理可按 segment_id 取。
