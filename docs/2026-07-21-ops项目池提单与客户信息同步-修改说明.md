# 2026-07-21 ops 项目池提单与客户信息同步修改说明

## 本次改动

1. 项目池支持直接提单
   - 项目名称列增加「+ 提单」入口。
   - 点击后复用需求提单的新建工单弹窗。
   - 自动带入当前项目和客户。
   - 未填写工单内容时关闭弹窗不再二次确认。

2. 协作成员列表支持按人提单
   - 协作成员弹框增加「提单」按钮。
   - 点击成员提单时自动带入当前项目和负责人。
   - 能匹配到环节时会自动带入环节。
   - 已禁用成员不显示提单入口。

3. 协作成员弹框优化
   - 成员按标签分组展示。
   - 默认展示「所有」。
   - 支持 tab 切换不同角色分组。
   - `unity开发` 和 `cocos开发` 合并到「开发」分组。
   - 已禁用标记改为红色。

4. 客户对接人与需求文档同步
   - 项目池支持修改客户对接人和需求文档。
   - 数据主存储仍在 soyoo 项目表。
   - ops 作为编辑和展示入口。
   - 修改后通过 soyoo 同步到飞书表格。
   - ops 项目池读取 snapshot / Redis 中的最新展示数据。

5. 项目池表格显示优化
   - 调整项目名称、客户名称、下版交付时间的显示宽度和字号。
   - 当前阶段和下版交付时间减少省略。
   - 项目名称列内容垂直居中。

## 数据流说明

### 项目池提单

项目池点击提单后，仍走原有 ops 工单创建接口。

流程：

1. 前端打开新建工单弹窗。
2. 自动填入当前项目。
3. 用户填写标题、环节、负责人和需求说明。
4. 后端创建工单。
5. 后端刷新项目池 snapshot。
6. 前端刷新项目池数据。

### 客户对接人 / 需求文档

流程：

1. ops 项目池修改客户对接人或需求文档。
2. ops 后端调用 soyoo integration 接口。
3. soyoo 更新项目字段。
4. soyoo 同步到飞书对应列。
5. ops 刷新项目池 snapshot / Redis。

## 注意事项

- Redis / snapshot 是加速缓存，不是最终数据源。
- 客户对接人和需求文档以 soyoo 项目表为准。
- 项目池提单复用需求提单弹窗，后续提单能力调整时两边会共同受影响。
- 协作成员分组只影响前端展示，不改变用户标签数据。
