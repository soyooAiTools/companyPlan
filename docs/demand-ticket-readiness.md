# 需求提单生产交付审计

日期：2026-06-12

## 结论

当前版本已经从前端 happy-path 原型升级为可真实承载业务数据的单节点生产数据系统：

- Node/Express API 提供真实数据写入接口。
- MySQL 持久化用户、项目、项目成员、提单、附件元数据、会话和审计日志。
- 附件文件落盘到服务端 `uploads/`。
- 种子演示附件也会生成真实文件，避免只有文件名和大小但无法打开。
- 登录使用服务端账号和 HttpOnly 会话 Cookie。
- 行级权限在服务端裁剪，前端不再通过账号切换模拟权限。
- 管理员可配置所属项目列表和各类型默认交付/风险小时；新建需求里的项目名称为用户手填文本。
- 提单创建、状态变更、附件上传、管理员配置、管理员甘特拖动/调整长度都会写审计日志。

它适合先以单节点部署承载项目组真实业务数据。进入多节点、高并发或跨公网企业级部署时，应按下方“后续增强”迁移到 PostgreSQL、对象存储、统一身份源和集中备份监控。

## 已补齐的生产环节

- 真实登录：`/api/auth/login`、`/api/auth/logout`、`/api/session`。
- 服务端数据入口：`/api/bootstrap` 只返回当前登录用户可见的数据。
- 服务端行级权限：管理员全量；其他账号只返回本人发起或本人负责的提单，同岗位/同类型人员不能互相看到无关提单。
- 服务端写入：新建提单、状态变更、管理员配置、甘特视觉偏移和视觉长度都写入数据库。
- 附件存储：图片、附件、文件上传时写入服务端目录并记录大小、类型、哈希、打开地址和下载地址。
- 审计日志：登录失败、登录、退出、建单、附件上传、状态变更、管理员配置、甘特拖动/调整长度均可记录。
- 小时制：提单时长、状态停留、剩余时间和风险判断都按小时计算。
- 静态资源生产服务：Node 服务同时提供 API 和 Vite 构建后的前端资源。
- 自动化生产场景测试：`npm run test:scenarios` 使用临时数据库启动真实生产服务。

## 权限矩阵

| 角色 | 导航 | 需求提单 | 延期任务预警 | 任务甘特图 | 甘特拖动 | 审计日志 |
| --- | --- | --- | --- | --- | --- | --- |
| 管理员 | 全部导航 | 全部提单 | 全部预警 | 全部提单 | 可以拖动/调整视觉时间线 | 可查看 |
| 项目负责人 | 仅需求提单 | 本人发起、本人负责 | 同范围 | 不可见 | 不可用 | 不可查看 |
| UI/美术/模型/动画等制作角色 | 仅需求提单 | 本人发起、本人负责 | 同范围 | 不可见 | 不可用 | 不可查看 |
| 程序员 | 仅需求提单 | 本人发起、本人负责 | 同范围 | 同范围只读 | 不可拖动 | 不可查看 |

## 已验证用例

运行命令：

```bash
npm run build
npm run test:scenarios
```

`test:scenarios` 会自动构建前端，在 `COMPANYPLAN_SCENARIO_PORT` 或 `4274` 启动隔离的生产 Node 服务，使用隔离 MySQL database 名称、临时上传目录和真实 Chromium 浏览器验证。只有显式设置 `COMPANY_PLAN_URL` 时才会测试已有服务：

- 未登录访问 `/api/bootstrap` 返回 `401`。
- 管理员通过真实用户名/密码登录后可见全局导航、三张底部工作表和新增工作表入口。
- 生产 UI 不再暴露账号切换下拉框。
- UI 设计、项目负责人、程序员看到的需求行、预警行、甘特行都没有越权。
- 模型账号顾远只能看到本人发起或指派给本人的需求，不会因为参与同项目、同为模型岗位而看到无关提单。
- 非程序员非管理员看不到 `任务甘特图`。
- 程序员能看到 scoped 甘特图，但甘特条只读且拖动无效。
- 管理员配置所属项目列表和模型等任务类型默认交付/风险小时后，新建提单会把所属项目下拉和用户手填项目名称分别保存。
- UI 设计创建真实提单，上传图片、附件、文件；刷新页面后提单仍存在。
- 附件详情可打开文件内容并触发浏览器下载。
- 初始种子提单的演示附件可直接打开，返回真实文件内容。
- 延期任务预警里的 `查看详情` 会切回 `需求提单` 工作表，并打开对应需求详情。
- 程序员能以“指派给我”看到该新提单。
- 需求表头半选态正常。
- 管理员拖动甘特条只改变该提单视觉偏移，调整右侧手柄只改变视觉长度，不改变行顺序和开始日期。
- 管理员更新后的视觉偏移和长度会同步出现在程序员 scoped 甘特图。
- 管理员可通过 `/api/audit` 查到建单、附件上传、管理员配置和甘特更新审计事件。
- 需求页工具栏只包含 `添加记录` 和 `查找`。
- 需求页不显示文档式顶栏、协作者/分享/顶部账号控件。
- 底部工作表标签固定在视口底部。

## 当前生产边界

- 当前实现是 Node/Express + MySQL + 本地磁盘附件，适合内网小团队和单实例服务部署。
- 生产数据必须备份 MySQL database 和 `uploads/`。
- 首次启动的种子密码必须通过 `COMPANYPLAN_SEED_PASSWORD` 改掉。
- 对公网或企业正式域名部署时必须使用 HTTPS，并设置 `COMPANYPLAN_COOKIE_SECURE=1`。

## 后续增强

- 多节点部署：保留 MySQL 托管/主从或迁移到 PostgreSQL，附件迁移到 S3/OSS/MinIO。
- 企业身份：接入 SSO/OIDC/LDAP，并支持账号禁用、强制改密和密码策略。
- 运维保障：增加健康检查探针、结构化日志、指标、备份恢复演练和容量告警。
- 数据治理：增加可配置角色、字段级审计、附件病毒扫描和保留策略。
