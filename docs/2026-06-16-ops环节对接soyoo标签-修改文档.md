## 四、关键设计(参考)
- **环节模型**:`ops_segments`(环节 + 交付/阈值)+ `ops_segment_tags`(环节 ↔ soyoo 标签)。环节是 ops 自定义分类,把 soyoo 标签绑进去;建单选环节 → 负责人按"该项目下标签属于该环节"的成员筛。
- **状态机**:排队中 → 进行中 →(阻塞,填原因)→ 已完成 →(重新打开)→ 进行中;**仅负责人/管理员**可改;完成/阻塞填备注;全程进 `ticket_events`。
- **可见性**:个人数据,owner 或 requester 是自己;管理员在此菜单也只看自己。
- **id 现状**:目录表沿用 soyoo 的 `ops-*` 前缀(同步副本);`ops_segments`/`ticket_events` 用 `INT` 自增;`tickets` 用 uuid。**干净 id 重设计推迟**。
