# 2026-07-17 ops 附件上传与在线预览修改说明

## 背景

需求提单的富文本说明里需要支持较大的 Word、Excel、PDF 等附件上传，并提供轻量在线查看能力。

原流程是前端把文件转成 base64 后提交给 ops 后端，再由后端上传 OSS。大文件会占用后端内存，也容易被请求体大小限制拦截。

## 本次改动

### 1. 附件上传改为 OSS 直传

新增后端接口：

- `POST /api/ops/upload-url`

前端上传流程改为：

1. 前端请求 ops 后端获取 OSS 签名上传地址。
2. 浏览器直接 `PUT` 文件到 OSS。
3. 上传完成后，把 OSS 文件地址写入富文本。

这样文件内容不再经过 ops 后端。

### 2. 文件大小限制

- 图片：仍限制 `2MB`
- 视频 / Word / Excel / PDF / 压缩包等附件：限制 `55MB`

默认配置在：

- `apps/server/config/runtime.mjs`
- `ossConfig.maxFileBytes`

也可以通过环境变量覆盖：

- `OSS_MAX_FILE_BYTES`

### 3. 上传进度

富文本编辑器内增加上传进度展示。

拖拽或选择附件后，会显示当前文件名和上传百分比。

### 4. 在线预览

接入 `jit-viewer`，新增预览页：

- `/file-preview`

上传后会根据文件名/MIME 判断是否可预览：

- PDF
- Word
- Excel
- PPT

富文本里只展示一个「查看」入口。

### 5. 大文件预览策略

为避免大文件在线预览过慢：

- `<= 20MB`：允许在线预览
- `> 20MB`：提单编辑器内不显示查看/下载入口，只保留文件名

预览页仍保留大文件兜底提示，用于兼容历史已经生成的预览链接。

### 6. 预览加载优化

`jit-viewer` 体积较大，已拆成懒加载：

- 普通进入 ops 系统时不加载预览 SDK
- 只有打开 `/file-preview` 时才加载

预览页又拆成两层：

- `FilePreviewPage`：轻量入口，只负责参数判断
- `JitFileViewer`：真正加载 `jit-viewer`

### 7. 预览下载进度

`jit-viewer` 不提供 Office 解析进度。

本次在进入 viewer 前，先用 XHR 下载 OSS 文件，并展示真实下载百分比：

- `文件下载中 37%`
- 下载完成后显示 `文件渲染中...`

渲染完成后隐藏进度层。

### 8. 富文本工具迁移

将富文本清洗工具从 ops 业务目录迁到公共 utils：

- 原位置：`apps/server/ops/rich-html.mjs`
- 新位置：`apps/server/utils/rich-html.mjs`

原因：富文本 sanitize、纯文本提取、空内容判断属于通用能力，不应放在 ops 业务目录下。

## 部署注意

本次同时涉及前端和 ops 后端。

部署时需要：

1. 重新构建前端。
2. 重启 ops 后端。
3. 确认 OSS Bucket 已允许浏览器直传需要的跨域规则。

OSS CORS 至少需要允许：

- `PUT`
- `GET`
- `HEAD`
- `OPTIONS`

允许请求头建议使用：

- `*`

暴露响应头建议包含：

- `ETag`

## 后续建议

当前附件信息仍然存储在富文本 HTML 中。

如果后面附件管理需求继续增加，建议新增附件表，单独存储：

- 工单 ID
- 文件名
- OSS URL
- MIME
- 文件大小
- 预览类型
- 上传人
- 上传时间

这样可以更方便地做权限、清理、统计和历史附件兼容。
